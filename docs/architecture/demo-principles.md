# Demo Principles

Purpose: keep the public demo aligned with Aurora's current operating message and with the Aurora OS boundary.

## Strategic message
- La decision profesional cambio
- Hoy equivocarse cuesta mas que contrastar
- antes: criterio, decidir y ajustar
- ahora: evidencia integrada, contrastar antes de mover

## Demo rule
- the demo is not a calculator
- the demo is not a prediction engine
- the demo is not a generic recommendation tool
- the demo is a controlled contrast layer and a structured evidence experience

## Interaction rule
- prefer semi-structured visible inputs over open-ended narrative input
- reveal one central vulnerability
- return one clear evidence-based direction
- keep the response compressed; no long explanations and no consultora language
- do not imply numeric precision promises beyond the Aurora OS response

## CTA rule
- challenge plus action, not passive viewing
- the site should make the user feel there is another standard of operating
