# Public Language Rules

Purpose: keep Aurora's public language institutional, structural, and consistent with the system canon.

## Required framing
- Decision Infrastructure
- evaluation before execution
- structure before action
- deterministic footprint
- end-to-end traceability
- system, not tool

## Canonical module labels
- Aurora Core
- Scenario
- Risk
- Signal
- Ledger

## Canonical solution labels
- Cost Optimization
- Growth Strategy
- Marketing Strategy
- Investment Evaluation
- Decision Support
- Cash / Runway Control

## Preferred terms
- structural load
- structural pressure
- pressure point
- compression mechanism
- decision footprint
- protocol
- run signature
- ledger
- trace layer

## Forbidden drift
- SaaS dashboard
- analytics platform
- generic AI recommendations
- more data
- optimization platform
- consultora

## Expression rule
- the site explains the system
- the demo proves the system
- neither surface should imply that the browser executes the Aurora core locally
