# Modules and Solutions Expression

Purpose: define how `aurora-site` expresses Aurora's canonical modules and solutions without duplicating system logic.

## Canonical modules shown publicly
- Aurora Core
- Scenario
- Risk
- Signal
- Ledger

## Canonical solutions shown publicly
- Cost Optimization
- Growth Strategy
- Marketing Strategy
- Investment Evaluation
- Decision Support
- Cash / Runway Control

## Site rule
- the homepage and demo may present modules and solutions as system surfaces
- the site may not simulate or redefine these modules in browser logic
- when a module or solution needs data, identity, pressure, or trace output, that output must come from Aurora OS

## Continuity rule
- home introduces the system and its module map
- demo turns public input into a request against Aurora OS canon
- results are rendered as proof of the system, not as an independent web engine
